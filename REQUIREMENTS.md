# x-logview 需求文档

## 项目概述

x-logview 是一个万能的跨平台日志查看工具，采用前后端分离架构，支持查看超大文件（GB级别），具备智能文件类型检测和编码识别功能。界面设计参考 Visual Studio Code。

## 功能需求

### 1. 界面设计
- 参考 Visual Studio Code 的界面设计和交互方式
- 采用现代化 UI 设计，支持主题切换
- 四栏布局：左侧边栏、主编辑区、右侧边栏、状态栏

### 2. 跨平台与远程支持
- 支持 Windows、macOS、Linux 跨平台运行
- 前后端分离架构
- 支持 VS Code 的 Remote 方式打开远程机器上的文件
- 支持 SSH、WSL 等远程连接方式

### 3. 超大文件支持
- 支持打开几个 GB 的超大文件
- 采用流式读取，只加载文件的一小部分到内存
- 可配置内存缓冲区的大小
- 不占用文件句柄，打开的文件可以继续修改/删除

### 4. 临时文件管理与状态恢复
- 新建临时文件自动保存到临时目录
- 打开时默认打开之前的临时文件继续编辑
- 恢复到关闭前的光标位置和视图状态
- 自动记录用户当前的文档位置，启动后恢复
- 自动保存用户未保存的修改到缓存中，启动后恢复
- 支持会话恢复功能

### 5. 性能要求
- 任何情况都不卡死前端程序
- 后端尽量异步执行，避免阻塞 UI 线程
- 采用流式处理，不等待整个文件加载完成

### 6. 文件类型检测
- 采样读取文件部分数据
- 自动判断文件是文本文件还是二进制文件
- 文本文件自动检测编码（UTF-8、GBK、ISO-8859-1 等）
- 二进制文件显示方式：
  - 左侧：16 进制显示
  - 右侧：可显示的文本字符

### 7. 编码转换
- 支持将文件从一种编码转换为另一种编码
- 支持批量编码转换
- 转换时保持文件内容完整性
- 支持预览转换结果

### 8. 流式文件加载与自动刷新
- 不需要等待文件完全读取即可打开查看
- 在不知道文件行数时就显示内容
- 滚动条可以不准确，未统计到的行号显示 "..."
- 在当前查看位置读完后再更新行号
- 支持动态加载和渲染
- 文件被修改后支持自动重新加载
- 重新加载不改变当前用户的位置等状态
- 支持正写入中的日志有 follow 模式，动态显示尾部的内容

### 9. 查找替换功能
- 支持基本的查找功能
- 支持替换功能
- 支持正则表达式
- 异步执行，不阻塞 UI
- 可以中断正在进行的查找/替换操作
- 支持高亮显示匹配结果

### 10. 文本高亮匹配
- 用户选中一段文本后
- 当前窗口中显示的相同文本都高亮显示
- 实时更新高亮，滚动时自动刷新

### 11. XML/JSON 处理
- 支持 XML/JSON 格式检查（语法错误提示）
- 支持 XML/JSON 自动格式化（便于阅读）
- 支持 XML/JSON 压缩（去掉多余换行和空格）
- 支持语法高亮显示

### 12. 数字智能提示
- 用户选中一个数字后
- 在主编辑区右侧的边栏窗口显示：
  - 该数字的 16 进制表示
  - 如果该数字是时间戳，显示对应的时间
- 支持多种时间戳格式（Unix 时间戳、毫秒时间戳等）

## 非功能需求

### 性能需求
- 文件打开响应时间 < 1 秒（无论文件大小）
- 内存使用可控，通过配置缓冲区大小限制
- UI 操作响应时间 < 100ms
- 支持 GB 级别文件的流畅滚动

### 兼容性需求
- 支持 Windows 10+
- 支持 macOS 10.15+
- 支持主流 Linux 发行版
- 支持高 DPI 显示屏
- 支持多种终端模拟器

### 可用性需求
- 界面直观，学习成本低
- 支持键盘快捷键
- 支持鼠标操作
- 提供详细的帮助文档

## 用户界面需求

### 主界面布局
```
┌─────────┬────────────────────────┬─────────┐
│         │                        │         │
│  左侧栏  │       主编辑区          │  右侧栏  │
│ (文件树) │      (日志内容)         │ (详情)  │
│         │                        │         │
├─────────┴────────────────────────┴─────────┤
│                    状态栏                    │
└─────────────────────────────────────────────┘
```

### 状态栏信息
- 文件编码（支持切换编码）
- 文件类型（文本/二进制）
- 当前行号/总行号（加载中显示 "..."）
- 文件大小
- 内存使用情况

## 技术约束

### 架构要求
- 前后端分离架构
- 后端：Go 语言
- 前端优先级：
  1. Electron + React/Vue（桌面端）
  2. Web 应用（浏览器端）
  3. 移动客户端（iOS/Android）
- 通信协议：WebSocket 或 gRPC

### 文件处理
- 流式文件读取
- 内存映射文件（可选）
- 异步 I/O 操作
- 智能缓存策略

### 编码支持
- UTF-8
- UTF-8 BOM
- UTF-16
- GBK/GB2312
- ISO-8859-1
- 自动检测编码
- 编码转换功能（支持将文件从一种编码转换为另一种编码）

## 验收标准

1. 能够打开 10GB+ 的日志文件，响应时间 < 1 秒
2. 打开文件后不占用文件句柄，文件可被其他程序修改/删除
3. 前端界面在任何操作下都保持响应
4. 二进制文件正确显示为十六进制+文本视图
5. 流式加载时滚动条和行号正确处理
6. 查找替换功能可中断，不阻塞 UI
7. 数字悬停提示正确显示十六进制和时间戳
8. 跨平台运行稳定，无平台特定 bug
